# Implementation Plan: PPO Agent

**Branch**: `001-ppo-agent` | **Date**: 2026-04-09 | **Spec**: [spec.md](./spec.md)

## Summary

Implement a complete PPO (Proximal Policy Optimization) agent in `@ignitionai/backend-tfjs` using TensorFlow.js. The agent uses an Actor-Critic architecture (shared trunk + 2 heads), collects rollouts, computes GAE advantages, and trains with a clipped surrogate objective. Implements AgentInterface from `@ignitionai/core`.

## Key Design Decisions

### Actor-Critic Architecture
- **Shared trunk**: MLP with configurable hidden layers (default [64, 64]) using ReLU
- **Policy head**: Dense → softmax (discrete action probabilities)
- **Value head**: Dense → linear (single scalar state value)
- Uses `tf.model()` (functional API) instead of `tf.sequential()` to support 2 outputs from 1 input

### RolloutBuffer (not ReplayBuffer)
- PPO is on-policy: data is collected, used for one round of training, then discarded
- Buffer stores parallel arrays: states[], actions[], rewards[], values[], logProbs[], dones[]
- After nSteps collected: compute GAE advantages + discounted returns, then clear

### Training Flow (inside AgentInterface contract)
1. `remember(exp)` — stores experience + current value/logProb in rollout buffer
2. `train()` — if buffer has < nSteps experiences: no-op (return immediately)
3. When buffer full: compute GAE → shuffle indices → train K epochs on mini-batches → clear buffer
4. Total loss = clipped_policy_loss + valueLossCoeff * value_loss - entropyCoeff * entropy

### PPOConfig Defaults
| Parameter | Default | Rationale |
|---|---|---|
| clipRatio | 0.2 | Standard PPO clip range |
| gaeLambda | 0.95 | Balance bias/variance in advantage estimation |
| entropyCoeff | 0.01 | Mild exploration encouragement |
| valueLossCoeff | 0.5 | Standard weighting |
| lr | 0.0003 | Lower than DQN for policy stability |
| nSteps | 128 | Rollout length before training |
| batchSize | 64 | Mini-batch size within each epoch |
| epochs | 4 | PPO epochs per rollout |
| gamma | 0.99 | Standard discount factor |
| hiddenLayers | [64, 64] | Sufficient for simple tasks |
