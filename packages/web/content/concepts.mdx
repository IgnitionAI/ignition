# Core Concepts

IgnitionAI is built around one idea: **describe your world as a class, and the framework handles the rest.**

## The `TrainingEnv` interface

Implement 5 methods and an `actions` property. That's your entire contract with the framework.

```ts
import { IgnitionEnvTFJS } from '@ignitionai/backend-tfjs'
import type { TrainingEnv } from '@ignitionai/core'

class MyGame implements TrainingEnv {
  actions = ['left', 'right', 'jump', 'shoot']

  observe(): number[] {
    return [
      player.x / WORLD_WIDTH,
      player.y / WORLD_HEIGHT,
      enemy.x / WORLD_WIDTH,
      enemy.y / WORLD_HEIGHT,
    ]
  }

  step(action: number): void {
    player.do(this.actions[action])
  }

  reward(): number {
    if (player.hitEnemy) return -10
    if (player.collectedCoin) return +5
    return -distance(player, nearestCoin) * 0.01
  }

  done(): boolean {
    return !player.alive || player.won
  }

  reset(): void {
    game.restart()
  }
}

const env = new IgnitionEnvTFJS(new MyGame())
env.train('dqn')
```

## Auto-configuration

The framework **deduces** the network shape from your env:

- `inputSize` ← length of the array returned by the first `observe()` call
- `actionSize` ← `actions.length`

You never touch neural network code.

## Tips for good results

- **Normalize observations** to `[-1, 1]` or `[0, 1]`. Neural networks hate unbounded inputs.
- **Shape your rewards.** Dense rewards (distance-based) converge faster than sparse rewards (goal-only).
- **Start simple.** Get DQN working on a small env before scaling up.
- **Let it run.** RL is slower than supervised learning. Be patient or crank `env.setSpeed(50)`.
