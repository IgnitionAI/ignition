# ONNX Export

The ONNX bridge is what makes IgnitionAI a serious tool, not a toy. Train in the browser, deploy everywhere.

## 1. Train in the browser

```ts
import { IgnitionEnvTFJS } from '@ignitionai/backend-tfjs'

const env = new IgnitionEnvTFJS(myEnv)
env.train('dqn')
// ... wait for convergence ...
env.stop()
```

## 2. Export to ONNX

```ts
import { saveForOnnxExport } from '@ignitionai/backend-onnx'

const { conversionScript } = await saveForOnnxExport(env.agent.model, './export')
```

## 3. Run the Python conversion script (one-time setup)

```bash
bash convert.sh
```

## 4. Deploy the `.onnx` model anywhere

- **Unity** via [Sentis](https://unity.com/products/sentis) or Barracuda
- **Unreal Engine** via [NNE](https://dev.epicgames.com/documentation/en-us/unreal-engine/nne-overview-in-unreal-engine)
- **Python / C++ / Rust** via [ONNX Runtime](https://onnxruntime.ai/)
- **Mobile / edge devices**

## Run inference directly in JS

You can also load the trained model back into a browser page for inference:

```ts
import { OnnxAgent } from '@ignitionai/backend-onnx'

const agent = new OnnxAgent({
  modelPath: './my-model.onnx',
  actionSize: 4,
})
await agent.load()
const action = await agent.getAction(observation)
```
